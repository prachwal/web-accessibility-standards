# WCAG 2.2

Primary source:
- W3C WAI: WCAG 2.2 Recommendation and supporting guidance
- https://www.w3.org/WAI/standards-guidelines/wcag/new-in-22/
- https://www.w3.org/WAI/news/2023-10-05/wcag22rec/

Use WCAG 2.2 as the current review baseline.

## High-value criteria to keep in mind

Structure and semantics:
- meaningful headings and labels
- correct lists, tables, forms, and landmarks
- reading order that matches intent

Keyboard and focus:
- all interactive controls operable by keyboard
- visible focus indicator
- focus not hidden behind sticky UI or overlays

Responsive and mobile-related:
- `1.3.4 Orientation`
- `1.4.10 Reflow`
- `1.4.12 Text Spacing`
- `2.5.8 Target Size (Minimum)`

Interaction-specific:
- `2.5.7 Dragging Movements`
- `2.4.11 Focus Not Obscured (Minimum)`
- `2.4.12 Focus Not Obscured (Enhanced)`
- `2.4.13 Focus Appearance`
- `3.3.8 Accessible Authentication (Minimum)`
- `3.3.9 Accessible Authentication (Enhanced)`

## Review posture

Map issues to specific success criteria when the task is compliance-oriented. For implementation tasks, use criteria as design constraints rather than only as a final checklist.
